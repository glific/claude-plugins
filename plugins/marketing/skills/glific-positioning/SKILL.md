# Glific Positioning Skill

**What Glific is:** The AI-powered WhatsApp chatbot platform built for nonprofits. We sell SaaS — lead with the platform and its benefits. The NGO is always the hero; Glific is the guide.

---

## Core Reference Card
*Everything needed for common tasks is here. Read reference files only when this card doesn't cover it.*

### Product descriptor
> India's Most Preferred WhatsApp Chatbot Platform for Nonprofits.

### Value spine
- **Primary — time freed:** Staff time consumed by manual WhatsApp operations returns to real program work: connecting with beneficiaries, improving programs, reporting.
- **Primary — reach at scale:** A program reaches thousands of beneficiaries in their language, at the time they need, without the reach being capped by what the team can do by hand.
- **Secondary — whole team runs it:** A non-technical program lead builds and runs flows themselves — not just the one tech person.
- **Secondary — two-way conversations:** WhatsApp lets programs converse with beneficiaries at scale — multilingual, voice, AI-driven context, every conversation becomes reportable data.

⚠ **Hard stop:** Never frame reach at scale as headcount reduction. Banned: "without growing the team," "without an extra hire," "do more with fewer people." The team matters more, not less — their time is just freed for better work.

### Differentiators (one per asset, never stacked)
- **D1 Built for nonprofits** — field-built sector templates, program language, not enterprise/e-commerce tooling
- **D2 Community of NGOs** — peer network (120+ NGOs), cohorts, sprints, meetups, co-creation. *One differentiator among several — not the headline. We sell SaaS.*
- **D3 AI personalisation** — context-aware, multilingual, voice flows; 1:1 feel at scale
- **D4 Open source / DPG** — NGO owns its data; DPDP + GDPR; Digital Public Good; BigQuery
- **D5 Transparent pricing** — one platform fee, no feature tiers, unlimited users; honest about usage layers

### Key proof stats
- **120+ NGOs** active across **10+ sectors** *(use 120+ for active; 150 only if team confirms "served to date")*
- **Civis:** 2,500 → 420,000 active respondents
- **TAP:** 10,000+ students reached weekly
- **MukkaMaar:** 3,000+ girls across 54 schools

### Brand lines
- *Tagline:* "Reach your beneficiaries where they are, when they need."
- *Value line:* "Glific helps nonprofit teams use WhatsApp effectively to reach their program goals at scale."
- *Community line (D2 assets only):* "When you join Glific, you join the nonprofits that came before you."
- *CTA:* "See what your program looks like on Glific."

### Personas
- **Operator** (program/M&E lead): *"I'm not a techie. How simple can you make this?"* → lead with D2 + time saved
- **Decision-Maker** (founder/ED): *"How do I take my program online at scale?"* → lead with reach at scale + peer proof

### Voice quick-check (Sage register)
- Program language: beneficiaries, interventions, outreach, funder reports, M&E — never "users" or "customers"
- Explain, don't perform. Measured connected sentences, not fragments fired for effect
- **Banned — narrative theater:** "The X was never the problem. The Y was:" / "It works — until it doesn't" / "What stayed with me was…" / "Here's the thing."
- **Banned — hype:** seamless, robust, transformative, game-changing, leverage (verb), cutting-edge
- **Banned — contrast:** "X, not Y" / "not just X but Y" — state the positive directly
- **Banned — staff-replacement:** any phrasing implying Glific replaces people or reduces headcount
- Never open a LinkedIn post with "Glific"
- Never invent a metric — use proof stats above or read `proof-library.md`

## Pre-task protocol
Before executing, do a 30-second internal pass: audience? channel? one idea? job to be done? Ask only if the answer is genuinely missing from the brief — never more than 3 questions.

- **Audit:** job of the page/post? primary audience? primary CTA?
- **New copy:** which primary benefit leads? audience? one desired action?
- **Campaign:** funnel stage? conversion goal? channel(s)?

If all three are clear, proceed without asking.

---

## Workflow
1. Run the pre-task protocol above.
2. Use the Core Reference Card for most tasks — check the routing table before loading any file.
3. One asset = one benefit + at most one differentiator + one proof point. No stacking.
4. Walk the traceability chain: positioning idea → problem → customer voice → proof → asset.
5. Draft consequence-led, short, specific, NGO as hero.
6. Validate before delivering (three checks below).

---

## Validation (three checks)
1. **Positioning:** names one lead benefit, one differentiator max, NGO is protagonist, community not the headline
2. **Voice:** Sage register, no banned phrases, no narrative theater, program language throughout; staff-replacement check — could any sentence imply fewer people needed?
3. **Proof:** every number in the card above or `references/proof-library.md`; quotes verbatim and attributed; one proof point only

*Could this have been written about Wati or Turn.io? If yes, rewrite — push for the specific NGO, number, program reality.*

---

## Output format
- **Internal docs / audits:** plain markdown, clarity over formatting, no styling
- **Audit deliverable:** one verdict line + three-column table only (current copy · suggested copy · why). Nothing else.
- **Website copy:** phone-first — short front-loaded lines, value in the first line
